## 목표
- 1차: SQL 1문제 + 알고리즘 2문제 이상
- 2차: SQL 1문제 + 알고리즘 1문제 이상 안정화
- 원칙: 문제 수보다 `복기`와 `재풀이 성공률`을 올린다.

## 전략
- SQL은 가장 안정적인 득점 구간이므로 매일 유지한다.
- 1차 전까지는 `구현 / 그리디 / BFS-DFS / DP 기초 / 백트래킹 기초 / 최단경로 기초`에 집중한다.
- 2차 전까지는 `DP / 그래프 / 백트래킹` 비중을 높인다.
- 지금 시점에서는 고급 주제에 욕심내지 않는다.

## 주차별 플랜

### 1주차 (3/9 ~ 3/15)
목표: SQL 감각 회복 + 1차용 기본 유형 정리

- SQL
  - 프로그래머스 SQL 고득점 Kit 1회독
  - `JOIN`, `GROUP BY`, `HAVING`, `STRING/DATE`, `NULL 처리` 정리
  - 쉬운 문제는 15~20분 안에 푸는 감각 만들기
- 알고리즘
  - 구현, 그리디, BFS, DFS, 시뮬레이션 위주
  - 난이도는 실버 1 ~ 골드 4 중심
  - 하루 2문제, 주간 10~12문제 목표
- 체크포인트
  - SQL 기본 유형을 다시 설명할 수 있어야 함
  - BFS/DFS/구현 문제를 풀이 없이 다시 풀 수 있어야 함

### 2주차 (3/16 ~ 3/22)
목표: 2차에도 이어지는 핵심 유형 확보

- SQL
  - 틀린 문제 재풀이
  - 중간 난도 문제를 20분 내 풀이 목표
- 알고리즘
  - DP 기초: 점화식 세우기, 1차원/2차원 DP, LIS/LCS/배낭 기초
  - 백트래킹: 순열, 조합, 부분집합, 방문 배열 관리
  - 그래프 기초: 다익스트라, 이진 탐색 응용 정도까지
  - 주간 10문제 내외, 대신 `풀이 이해 후 재구현`까지 포함
- 체크포인트
  - DP 문제를 보고 상태 정의를 말로 설명할 수 있어야 함
  - 백트래킹에서 가지치기 조건을 직접 설계할 수 있어야 함

### 3주차 (3/23 ~ 3/27)
목표: 실전 운영과 시간 배분 고정

- 120분 모의고사 3회 이상
- 매 모의마다 SQL을 먼저 풀고, 남은 시간에 알고리즘 2문제 공략
- 모의 후 반드시 확인할 것
  - 첫 20분 안에 무엇을 풀었는지
  - 막힌 문제를 몇 분째 버렸는지
  - 실수인지, 개념 부족인지, 구현 체력 문제인지
- 약점 보강
  - 자주 틀리는 유형 1~2개만 골라 집중
  - 새 유형 확장보다 기존 약점 제거가 우선
