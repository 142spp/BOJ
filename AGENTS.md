# AGENTS.md

이 문서는 `/home/142spp/workspace/boj`에서 작업하는 에이전트용 기본 지침이다.

## 1) 목표
- 이 저장소는 코딩테스트(PS) 풀이 기록 저장소다.
- 최우선은 `정답성`, 다음은 `가독성`, 그 다음은 `성능`이다.

## 2) 기본 작업 원칙
- 기존 풀이를 임의로 지우거나 대규모 리팩터링하지 않는다.
- 요청이 없으면 문제별 기존 코드 스타일을 유지한다.
- 변경 범위는 최소화한다.
- 설명이 필요할 때는 한국어로 짧고 명확하게 작성한다.

## 3) 파일/폴더 규칙
- 풀이 코드는 기본적으로 `codes/` 하위에 둔다.
- 원본을 유지해야 할 때 최적화 버전은 `*_opt.cpp`로 만든다.
- 문제 번호/플랫폼 규칙이 이미 존재하면 그 규칙을 우선 따른다.

## 4) C++ 작성 규칙
- 언어 표준은 기존 파일/환경을 따르되, 충돌이 없으면 C++17 기준으로 작성한다.
- 입출력 성능을 위해 필요 시 아래 설정을 사용한다.
  - `ios::sync_with_stdio(false);`
  - `cin.tie(nullptr);`
- 오버플로우 가능성이 있으면 `long long`을 우선 고려한다.
- 매직 넘버는 상수로 분리한다.

## 5) 문제 해결 워크플로우
1. 문제 조건/입출력/제한을 먼저 요약한다.
2. 시간복잡도 목표를 제한에 맞춰 정한다.
3. 자료구조/알고리즘을 선택하고 근거를 남긴다.
4. 구현 후 최소 1개 이상 예제로 검증한다.
5. 필요한 경우 반례를 추가한다.

## 6) 검증 규칙
- 가능하면 컴파일 확인 후 결과를 공유한다.
- 기본 컴파일 예시:
  - `g++ -std=c++17 -O2 -Wall -Wextra -o main <file>.cpp`
- 실행 예시:
  - `./main < input.txt`

## 7) 기록 규칙
- 요청 시 `PS.md`에 아래 항목을 간단히 남긴다.
  - 문제/링크
  - 핵심 아이디어
  - 시간복잡도
  - 실수 포인트/반례

## 8) 금지 사항
- 정답이 불확실한 상태로 완료 처리하지 않는다.
- 관련 없는 파일을 함께 수정하지 않는다.
- 사용자 요청 없이 기존 정답 코드를 다른 방식으로 전면 교체하지 않는다.
